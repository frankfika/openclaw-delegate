## 一、核心问题与解决方案

### 1.1 我们解决什么问题？

**DAO治理的现状是一团糟：**
- **碎片化**：1000+ DAO分布在50+条链，用户需要在Snapshot、Tally、Discourse、Discord之间来回切换
- **低参与率**：平均投票参与率仅3-10%，Uniswap治理投票率经常低于5%
- **信息过载**：一个提案动辄几千字，普通用户根本不知道该怎么投
- **没有即时激励**：投票是义务劳动，除了"对协议有好处"外，用户得不到任何即时回报

### 1.2 我们的解决方案

VoteNow = **治理聚合器 + AI分析师 + 激励层**

**核心功能：**
1. **一站式投票**：近20个主流DAO的提案全部聚合在一个界面，支持Snapshot链下签名投票 + OpenZeppelin Governor链上投票，一键完成
2. **AI分析师（Governance Agent）**：DeepSeek驱动的专属分析师，用2句话告诉你提案是什么、风险多大、该怎么投
3. **投票即赚**：每次投票获得40-100基础积分（按DAO级别分层），早投奖励+20%、连续投票最高+50%，积分可兑换USDC、NFT、gas券
4. **智能提醒**：Telegram Bot集成，关注的DAO有新提案时自动推送，支持Bot内直接投票

## 二、AI 如何创造价值？当前能力与演进路线

### 2.1 当前已实现：AI 治理分析引擎

VoteNow 的 AI 分析基于 DeepSeek 大语言模型，针对 DAO 治理场景做了专项 Prompt 工程优化。当前已上线并稳定运行的能力：

| 功能 | 实现方式 | 用户价值 |
|------|---------|---------|
| **提案摘要** | LLM 提取提案核心内容，生成2句话概要 | 30秒读懂一个提案，替代阅读几千字原文 |
| **四级风险评估** | 基于提案类型（参数调整/资金转移/合约升级/治理机制变更）输出 Low/Medium/High/Critical | 快速识别需要重点关注的提案 |
| **策略匹配评分** | 根据用户风险偏好，给出0-100的匹配度 | 帮助用户快速筛选与自己策略一致的提案 |
| **投票建议** | For/Against/Abstain + 理由 | 降低决策门槛，尤其对新手有价值 |
| **安全检查** | 检查资金影响、合约变更、权限变更三个维度 | 降低恶意提案风险 |
| **对话式问答** | 多轮对话Agent，用户可以就提案内容追问，AI 基于提案上下文回答 | 深度研究不需要离开平台 |

**技术实现**：
- 模型：DeepSeek-chat，通过 OpenAI SDK 兼容接口调用
- 分析管线：提案文本 → 结构化 Prompt → LLM → JSON 解析 → 字段校验 → 展示
- 容错：AI 不可用时自动降级为"建议人工审阅"，不会阻塞投票流程
- 成本：DeepSeek 调用成本约为 GPT-4 的 1/20，具备规模化条件

### 2.2 技术架构：当前 vs 演进

#### 当前架构（已上线）

```
用户提交提案分析请求
    │
    ▼
Hono API 后端（Node.js，支持Cloudflare Workers/Railway部署）
    │
    ├─ 提案数据 ← Snapshot GraphQL API
    │
    ├─ AI 分析 ← DeepSeek API（结构化 Prompt）
    │
    ├─ Snapshot投票 ← Snapshot SDK（EIP-712 签名，用户浏览器端完成）
    │
    └─ 链上投票 ← OpenZeppelin Governor合约（通过wagmi/viem，用户浏览器端完成）
```

**这是一个清晰的三层结构**：数据获取 → AI 分析 → 用户操作。当前够用，且每一层都有明确的扩展路径。

#### 演进路线：从单模型分析到多维度决策支持

**Phase 1（当前）：单模型 + 结构化 Prompt**
- DeepSeek 做提案文本分析
- 固定 Prompt 模板，人工迭代优化
- 适合当前规模（19个 DAO，数百个活跃提案）

**Phase 2：数据增强分析**
- 将链上数据（国库余额、历史投票分布、提案通过率）作为上下文注入 Prompt
- 加入提案历史对比：类似提案过去的结果如何
- 数据来源仍然是 Snapshot API + 公开链上数据，不需要额外基础设施

**Phase 3：多步骤分析管线**
- 将分析拆分为独立步骤：提案分类 → 风险评估 → 策略匹配 → 综合建议
- 每个步骤用针对性 Prompt，提升各环节准确率
- 对比单次调用，多步骤管线能更好地处理复杂提案（如涉及多个合约变更的提案）

**Phase 4：用户行为驱动的个性化**
- 记录用户实际投票选择 vs AI 建议的偏差
- 用偏差数据调整 Prompt 中的策略权重（不需要模型微调，Prompt 层面即可实现）
- 例：某用户总是对"资金转移类"提案投 Against → 自动调高该类提案的风险权重

### 2.3 与 OpenClaw 的关系

**OpenClaw（https://openclaw.ai/）是我们团队开发的开源个人 AI 助手项目。**

VoteNow 的 AI 分析能力和 OpenClaw 共享同一个团队的 AI 工程经验，但两者是独立产品：

- **VoteNow** 是 Web 应用，面向治理投票场景，强调聚合和易用
- **OpenClaw** 是本地 AI 助手，面向通用个人效率场景

**协同价值**：
- OpenClaw 积累的 Prompt 工程、上下文管理、工具调用经验，直接服务于 VoteNow 的 AI 分析模块
- 未来 VoteNow 的治理分析能力可以作为 OpenClaw 的一个 Skill 插件，让 OpenClaw 用户在本地环境中使用治理分析
- 团队在 AI 应用层有双线实践，对 LLM 的能力边界和工程化有实战认知

### 2.4 为什么 AI 是差异化而非噱头？

| 维度 | VoteNow | 竞品（Snapshot/Tally/Boardroom） |
|------|---------|-------------------------------|
| **提案理解** | AI 自动摘要 + 风险分级 | 用户自己读原文 |
| **决策辅助** | 策略匹配 + 投票建议 | 无 |
| **深度研究** | 对话式追问 | 需要去论坛/Discord 问 |
| **成本结构** | DeepSeek，成本可控 | 不适用 |
| **迭代速度** | Prompt 迭代，无需重新训练 | 不适用 |

**核心壁垒不是"有AI"，而是**：
1. **治理场景的 Prompt 资产** —— 针对 DAO 提案的分析模板经过持续优化，通用 LLM 直接用效果差很多
2. **结构化输出工程** —— 让 LLM 稳定输出可用的 JSON（风险等级、匹配分数、安全检查项），需要大量的格式约束和容错处理
3. **场景理解** —— 知道治理提案里什么信息重要（资金规模、合约变更、权限调整），这是领域知识，不是通用能力

## 三、数据来源：100%真实，直连链上

### 3.1 数据从哪里来？

**不是爬虫，是官方API直连：**

| 数据源 | 获取方式 | 实时性 | 可靠性 |
|--------|---------|--------|--------|
| **提案数据** | Snapshot GraphQL API（hub.snapshot.org/graphql） | 实时 | Snapshot是行业标准，Uniswap/Aave等都用它 |
| **投票权力** | Snapshot API 查询（基于链上快照区块的token余额） | 实时 | 由Snapshot节点读取链上状态 |
| **Snapshot投票** | EIP-712签名（用户浏览器端通过MetaMask完成） + 提交到Snapshot Hub | 实时 | 签名直接提交到Snapshot官方节点 |
| **链上投票** | OpenZeppelin Governor合约调用（通过wagmi/viem） | 实时 | 链上交易，不可篡改 |
| **AI分析** | DeepSeek API（通过OpenAI兼容接口） | 按需 | 内置降级策略，AI不可用不影响投票 |
| **用户积分** | 服务端内存存储（当前阶段），后续迁移至持久化数据库 | 实时 | 投票成功后即时发放 |

### 3.2 为什么大家能在这里Vote？

**我们不是"替代"底层治理设施，而是在其之上做增强层：**

**Snapshot投票流程（80%的DAO）：**
1. 用户在VoteNow选择投票选项
2. 生成EIP-712结构化消息
3. 用户用MetaMask签名（off-chain，无gas费）
4. 签名提交到Snapshot Hub（hub.snapshot.org）
5. Snapshot验证签名并记录投票
6. 投票结果在Snapshot官方页面可见

**链上投票流程（Aave/Uniswap/Compound/ENS/Lido）：**
1. 用户在VoteNow选择投票选项
2. 调用对应Governor合约的`castVote`方法
3. 用户用MetaMask确认交易（需付gas费）
4. 投票记录上链，不可篡改

**换句话说**：在VoteNow投票 = 在官方页面投票，但聚合了多DAO、AI分析、积分激励。

### 3.3 Snapshot vs 链上治理：科普澄清

**这个问题很常见，甚至很多Web3从业者也混淆：**

#### 两种DAO治理机制

| 维度 | Snapshot（链下签名） | 链上治理（On-Chain） |
|------|---------------------|---------------------|
| **提案是什么** | JSON数据（标题、描述、选项） | 智能合约调用代码 |
| **存储位置** | Snapshot Hub服务器 | 区块链上（不可篡改） |
| **投票成本** | 0 gas（免费） | $5-50 gas/次 |
| **执行方式** | 多签钱包人工执行 | 合约自动执行 |
| **使用比例** | **80%的DAO** | 大型协议（Aave、Compound） |
| **示例** | Arbitrum、Gitcoin、ParaSwap | Aave、Compound |

> **注**：部分DAO同时使用两种机制。例如Uniswap、ENS、Lido均使用Snapshot做社区温度投票（temperature check），链上Governor做最终执行投票。VoteNow两种机制均已支持。

#### 为什么以Snapshot为主、链上为辅？

**市场原因**：
- **80%的DAO治理**使用Snapshot（链下签名），这是最大的存量市场
- 用户习惯了免费投票，不愿意付gas费
- 大多数提案不需要立即执行（社区信号先行）

**技术原因**：
- Snapshot API成熟、稳定，可以快速覆盖大量DAO
- 链上Governor合约已支持（Aave、Uniswap、Compound、ENS、Lido），但每个协议需单独适配
- 先用Snapshot验证产品-market fit，同时逐步扩展链上覆盖

#### 路线图：持续扩展治理覆盖

```
Phase 1（当前）: Snapshot + 基础链上治理
    ↓ Snapshot投票：覆盖全部19个DAO
    ↓ 链上投票：已支持OpenZeppelin Governor（Aave/Uniswap/Compound/ENS/Lido）
    ↓ 覆盖Ethereum/Arbitrum/Optimism/Polygon 四条主链
Phase 2: 深度链上集成
    ↓ 接入更多Governor框架（Nouns等）
    ↓ 链上提案执行状态追踪
    ↓ 扩展至Base/zkSync/BSC
Phase 3: 全链治理基础设施
    ↓ 覆盖所有主流治理框架
```

#### AI 安全分析的价值

**Snapshot的风险**：**文字游戏**
```
提案写："优化协议参数，提升用户体验"
实际：人工执行时，多签把钱转走
```
**我们的AI分析**：从提案文本中识别大额资金转移、异常权限变更等风险信号，给出风险等级

**链上的风险**：**恶意提案**
```
提案包含合约升级调用
结果：自动执行，可能影响协议安全
```
**我们的AI分析**：分析提案描述中涉及的合约变更，标记高风险操作（如合约升级、大额转账）

**说明**：当前AI安全分析基于提案文本内容，属于"风险信号识别"而非"合约代码审计"。对于Snapshot链下提案（占80%），文本分析已经覆盖主要风险场景。链上合约级别的静态分析是未来演进方向。

## 五、不仅仅是投票：DeFi + DID + 空投的基础设施

### 5.1 我们积累的资产：用户行为数据

每投一票，我们记录：
- 用户钱包地址
- 投票的DAO和提案
- 投票选择（For/Against/Abstain）
- 投票时间和所属链
- 投票权重（Snapshot投票由快照区块的token余额确定，链上投票由Governor合约的`getVotes`确定）

**这等于什么？**
= 用户的链上治理简历
= 真实的Web3参与度证明
= 比单纯持有代币更有价值的DID数据

### 5.2 数据应用场景

#### A. DeFi积分/空投凭证
未来项目方空投不再只看"你有多少币"，而是看"你有没有参与过治理"。

VoteNow可以：
- 为用户生成"治理分数"（类似芝麻信用）
- 项目方用我们的API查询真实参与者
- 我们成为"治理即服务"(Governance-as-a-Service)基础设施

#### B. DID身份层
- 投票记录 = 链上信用历史
- 高参与度用户 = 优质Web3公民
- 可以用于：无抵押借贷、DAO入职、白名单筛选

#### C. 定制化空投
项目方可以通过我们：
- 向"参与过Aave治理的用户"空投
- 向"总是投For的积极用户"空投
- 向"治理分数>80的用户"空投

#### D. VoteNow积分代币化（$VOTE Token）

**类似Solana手机的模式**：
- 早期用户通过投票积累积分
- 积分未来可按比例兑换$VOTE代币
- 代币价值来源于平台手续费回购、治理权

**代币经济模型**：
| 用途 | 比例 | 说明 |
|------|------|------|
| 社区空投 | 40% | 按历史积分分配给早期投票用户 |
| 生态激励 | 30% | 持续的投票奖励、流动性挖矿 |
| 团队 | 20% | 4年解锁 |
| 国库 | 10% |  DAO治理储备 |

**代币价值捕获**：
- 手续费回购：平台收入的50%用于回购$VOTE
- 治理权：$VOTE持有者可投票决定积分发放规则、DAO上线优先级
- 质押收益：质押$VOTE获得平台手续费分红

> $VOTE代币的发行链选择与HashKey Chain战略协同，详见第七章7.3节。

## 六、竞争分析：对标是谁？为什么我们更好？

### 6.1 直接竞品

| 竞品 | 他们是什么 | 他们的弱点 | 我们的优势 |
|------|-----------|-----------|-----------|
| **Snapshot** | 治理基础设施（我们依赖它） | 只有一个一个DAO看，没聚合，没AI，没激励 | 在Snapshot之上做聚合层 + 价值增值 |
| **Tally** | 链上治理聚合 | 只支持链上治理（错过80%的Snapshot投票），界面老旧，没AI | Snapshot + 链上双轨支持 + AI分析 + 激励 |
| **Boardroom** | 治理聚合器 | 只能看，不能投，没奖励，数据滞后 | 原生投票 + 即时激励 |
| **DeepDAO** | DAO数据分析 | 纯数据展示，没参与工具 | 参与导向，投票即可赚 |
| **Karma** | 委托代理评分 | 只服务委托人，普通用户用不上 | 面向所有持币者，积分系统 |

### 6.2 为什么我们比他们好？

**唯一一个同时具备以下三者的产品：**
1. ✅ 多链多DAO聚合（当前近20个 DAO，目标覆盖90%的主流治理）
2. ✅ AI驱动的决策支持（不是简单的总结）
3. ✅ 即时激励机制（积分可兑换真实价值）

**技术壁垒**：
- 治理场景专项 Prompt 工程积累（通用 LLM 直接用于治理分析效果差）

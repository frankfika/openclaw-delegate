## 一、核心问题与解决方案

### 1.1 我们解决什么问题？

**DAO治理的现状是一团糟：**
- **碎片化**：1000+ DAO分布在50+条链，用户需要在Snapshot、Tally、Discourse、Discord之间来回切换
- **低参与率**：平均投票参与率仅3-10%，Uniswap治理投票率经常低于5%
- **信息过载**：一个提案动辄几千字，普通用户根本不知道该怎么投
- **没有即时激励**：投票是义务劳动，除了"对协议有好处"外，用户得不到任何即时回报

### 1.2 我们的解决方案

VoteNow = **治理聚合器 + AI分析师 + 激励层**

**核心功能：**
1. **一站式投票**：近20个主流DAO的提案全部聚合在一个界面，支持Snapshot链下签名投票 + OpenZeppelin Governor链上投票，一键完成
2. **AI分析师（Governance Agent）**：DeepSeek驱动的专属分析师，用2句话告诉你提案是什么、风险多大、该怎么投
3. **投票即赚**：每次投票获得40-100基础积分（按DAO级别分层），早投奖励+20%、连续投票最高+50%，积分可兑换USDC、NFT、gas券
4. **智能提醒**：Telegram Bot集成，关注的DAO有新提案时自动推送，支持Bot内查看分析并跳转投票

## 三、AI + Web3 技术架构

### 3.1 当前已上线

**AI 治理分析引擎**：基于 DeepSeek，针对 DAO 治理场景做专项 Prompt 优化

| 功能 | 用户价值 |
|------|---------|
| 提案摘要 | 30秒读懂提案，替代读几千字原文 |
| 四级风险评估 | Low/Medium/High/Critical，快速识别风险 |
| 投票建议 | For/Against/Abstain + 理由 |
| 对话式追问 | 深度研究无需离开平台 |

**Web3 治理基础设施**：官方 API 直连，100% 真实数据

| 数据源 | 获取方式 |
|--------|---------|
| 提案数据 | Snapshot GraphQL API |
| Snapshot投票 | EIP-712签名（0 gas） |
| 链上投票 | OpenZeppelin Governor（Aave/Uniswap/Compound/Arbitrum） |

**架构**：
```
提案数据 ← Snapshot GraphQL API
    │
    ├─ AI 分析 ← DeepSeek API
    │
    └─ 用户投票 ← EIP-712签名 / Governor合约
```

### 3.2 演进方向：自主治理 Agent

借鉴 OpenClaw 架构，打造 DAO 版的自主 Agent：

| 维度 | 当前 | 未来 |
|------|------|------|
| 触发方式 | 用户主动查看 | Agent 自动监控推送 |
| 决策模式 | AI 建议，用户投票 | 学习偏好，半自主执行 |
| 记忆能力 | 无状态 | 持久化用户偏好历史 |
| 接入渠道 | Web | Telegram/Discord/WhatsApp |

**四层架构**：
1. 感知层：持续监控提案流
2. 分析层：DeepSeek 驱动的多步骤管线
3. 执行层：EIP-712 / 合约调用，支持预授权
4. 安全层：人类最终决策权
